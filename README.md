Color_Sort Explanation:
The problem gives us three different numbers: 0, 1, and 2. These numbers represent the colors and they're given to us in an array. Our approach to the problem was to do a count sort. We first initialized three different counters for each number (zerocount, onecount, twocount). Then we iterated through the array and counted how many instances of each integer appeared. After we had that, we changed the original nums array. We started at index 0 and began replacing the numbers in the original array (i.e, if original array was [1, 0], it would rewrite the "1" to a 0), and then we did that for each number in order of zero, one, two. This wouldn't change the size of the array since all the counts add up to the size of the array. The time complexity of this algorithm is O(n) because we make one pass through the array to count the elements and another pass to overwrite the array, both of which are linear. The space complexity is O(1) since we only use three integer counters regardless of the input size.

Maximum Gap Explanation:
To achieve linear time complexity, we cannot sort the array directly because sorting takes O(nlog⁡n)O(n \log n)O(nlogn). Instead, we use a bucket-based approach inspired by the Pigeonhole Principle. First, we find the minimum and maximum values in the array to determine the overall range. If there are nnn elements, then the minimum possible maximum gap between consecutive sorted elements is (max−min)/(n−1)(\text{max} - \text{min}) / (n - 1)(max−min)/(n−1). Using this idea, we divide the range into buckets of at least this size and place each number into its appropriate bucket, keeping track of only the minimum and maximum value within each bucket. Since numbers within the same bucket cannot produce a gap larger than the bucket size, the true maximum gap must occur between the maximum of one non-empty bucket and the minimum of the next non-empty bucket. By scanning through the buckets once to compute these differences, we obtain the maximum successive gap in overall O(n)O(n)O(n) time and O(n)O(n)O(n) extra space.
